# 広島県居住者・出身者 可視化マップアプリ「ひろしまの和」

「広島県出身です」と聞いただけでは、広島市なのか、福山市なのか、呉市なのか、県北なのか、島しょ部なのかが直感的に分かりにくいという課題を解決するために作られたWebアプリケーションです。

実際の正確な行政区域境界（GeoJSON）データと、美しいコラプレスマップ（塗り分け地図）を用い、登録者の居住地・出身地が広島県内のどこに集中しているのかを一目で分かりやすく可視化します。

---

## 🚀 アプリの概要と特長

1. **100% 正確な広島県の行政区域境界**:
   - 国土数値情報の公式な行政境界データ（JISコード `34`：広島県）を収集・最適化し、正確な地形と市町村境界を表示します。
2. **2つの表示モード**:
   - **市町村モード**: 広島県全体の23市町を表示します。広島市は1つの統合された大きな領域として塗り分け・連動します。
   - **広島市区モード**: 広島市をクローズアップし、8区を正確な区境で表示します。
3. **登録人数による自動カラー塗り分け（コラプレスマップ）**:
   - 0人：暗いグレー (`#1e293b`)
   - 1人：薄い青
   - 2〜3人：中くらいの青
   - 4人以上：濃い青
4. **ホバー・クリックによる双方向インタラクション**:
   - 地図上の市町村・区をホバーすると境界線が黄色に強調され、ツールチップで地域名が浮かび上がります。
   - クリックすると、登録されたメンバー名一覧が美しい吹き出しポップアップでその場に表示され、左側アコーディオンリストが自動スクロール連動します。
5. **便利な管理機能**:
   - CSV出力（BOM付きでExcelでも文字化けしません）
   - データの個別削除および一括クリア
6. **CORS対策・画面上エラー表示の搭載**:
   - ローカル環境で `index.html` を直接ダブルクリックで開いた場合などに、ブラウザのセキュリティ制限（CORS）で GeoJSON の読み込みが拒否される問題を検知し、地図コンテナ上に親切な案内カードを表示します。

## 💻 ローカルでの起動方法

ブラウザのセキュリティ制限（CORS）により、`fetch` を使用してローカルの GeoJSON ファイルを読み込む際、ダブルクリックで直接 `index.html` を開くとブラウザが読み込みを拒否し、地図が表示されなくなります。

ローカル環境で動作させるには、以下のいずれかの簡易Webサーバーを起動してアクセスしてください。

### 🐍 Python を使う場合 (最も簡単)
コマンドプロンプトやターミナルで `hiroshima-map` ディレクトリに移動し、以下を実行します：
```bash
python -m http.server 8000
```
起動後、ブラウザで以下のURLにアクセスしてください：
👉 [http://localhost:8000](http://localhost:8000)

### 📦 Node.js を使う場合
npm パッケージの `serve` を使用して起動します：
```bash
npx serve .
```
起動後、ターミナルに表示されるURL（通常は `http://localhost:3000` や `http://localhost:5000`）にアクセスしてください。

## 🌐 Web上への公開・デプロイ方法

本アプリはHTML / CSS / JavaScript / GeoJSON のみで構成された**完全な静的サイト**であるため、以下の静的ホスティングサービスを使用することで、**誰でもアクセスできるパブリックなWebアプリ**として全世界に完全無料で公開できます。

### 🐙 1. GitHub Pages での公開手順（推奨）
1. GitHubに新しいパブリックリポジトリを作成します。
2. 作成したリポジトリに、`index.html`、`style.css`、`script.js`、`README.md`、および `data/` フォルダを丸ごと push します。
3. リポジトリの **Settings**（設定）タブを開きます。
4. 左サイドバーから **Pages** を選択します。
5. Build and deployment の **Source** で **"Deploy from a branch"** を選択します。
6. **Branch** で `main` (または `master`) ブランチを選択し、フォルダを `/ (root)` に設定して **Save**（保存）を押します。
7. 数分後、ページ上部にデプロイ完了通知と公開URL（例：`https://[ユーザー名].github.io/[リポジトリ名]/`）が表示されます。そのURLにアクセスすれば、誰でもアプリを閲覧・利用できるようになります。

### ⚡ 2. Vercel での公開手順
1. [Vercel](https://vercel.com/) にログイン（またはGitHub連携で新規登録）します。
2. ダッシュボードで **"Add New" ➔ "Project"** を選択します。
3. アプリを push した GitHub リポジトリを **Import** します。
4. Configure Project 画面で、以下のように設定します：
   - **Framework Preset**: `Other`（または `Static Site`）を選択します。
   - **Build Command**: 空欄（未入力）のままで構いません。
   - **Output Directory**: 空欄のままにするか、`.`（ルート）を指定します。
5. **Deploy** ボタンを押します。
6. 数秒でビルドが完了し、`https://[プロジェクト名].vercel.app` のような公開URLが自動発行されます。

## ⚠️ 登録データの保存仕様と注意点

* **ブラウザ（localStorage）単位での保存**:
  - 本アプリに登録されたお名前や居住地データは、**各ユーザーのブラウザの `localStorage` にのみ保存**されます。
  - そのため、Aさんが登録した内容が、Bさんのブラウザや他人のPC画面にリアルタイムに共有されることはありません。
  - 公開デプロイされた同一のURLに全員でアクセスした場合でも、登録データはアクセスした個人個人の端末内にのみ独立して保持されます。

## 🚀 将来的なバックエンド拡張案

将来的に「みんなが登録したお名前や居住地データをリアルタイムに1つのデータベースに集約・集計し、全員で同じ地図の塗り分け画面を共有したい」場合は、以下の拡張を行ってください。

1. **Firebase / Supabase などの共有データベース連携**:
   - `script.js` の `saveCurrentData()` の内容を、クラウド上のデータベース（Firebase Realtime Database / Firestore / Supabase など）へ非同期POSTリクエストを送信してクラウド上に保存するロジックに置き換えます。
   - `loadAllData()` の内容を、アプリ起動時にクラウドAPIから全データをGETリクエストで取得する非同期通信に置き換えます。
   *(※ 詳細な書き換えのコード例は、`script.js` の `8. localStorage 連携 & 将来的な共有データベース化について` のコメントブロック内に記載されています)*
2. **Google Sheets API の活用**:
   - Google スプレッドシートをデータベース代わりに使い、登録データをスプレッドシートへ書き出すことで、ノンプログラミングでデータ蓄積と管理者画面の構築を両立させることも可能です。
3. **管理者機能の追加**:
   - 不適切な名前や登録内容を管理者が削除できる「管理用パスワード付きのメンテナンス画面」を追加する。
4. **CSVインポート機能**:
   - 既存の名簿データを一括でマップに流し込めるよう、CSVファイルをインポート（読み込み・パース）して一括で塗り分けに反映させる機能の追加。
